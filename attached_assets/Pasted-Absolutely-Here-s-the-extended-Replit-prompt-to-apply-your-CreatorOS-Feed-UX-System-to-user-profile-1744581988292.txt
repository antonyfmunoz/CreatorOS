Replit Prompt: CreatorOS Profile Page with Integrated Feed (Inspired by X)

Extend the CreatorOS UI system to include a mobile-first, modern Profile Page for each user. This page should reuse your existing post card system (including voice posts, filters, etc.) and display the user’s content in a scrollable feed under a sleek profile header, styled similarly to X (Twitter). The Floating Action Button (FAB), dark mode compatibility, and filtering should remain consistent.

⸻

1. Sticky Profile Header (X-Inspired)

<Header className="sticky top-0 z-50 bg-white dark:bg-black px-4 py-2 shadow-sm">
  <span className="text-xl font-semibold text-black dark:text-white">{user.name}</span>
</Header>

	•	Shows user’s name
	•	Optionally add a back button
	•	Always sticky during scroll

⸻

2. Profile Info Section (Top of Feed)

<ProfileHeader>
  <div className="flex flex-col items-center py-4">
    <img src={user.profileImage} className="w-24 h-24 rounded-full object-cover" />
    <h2 className="font-semibold text-lg">{user.name}</h2>
    <p className="text-gray-500">@{user.username}</p>
    <p className="text-sm mt-2 whitespace-pre-line">{user.bio}</p>
    {user.links.length > 0 && (
      <div className="mt-2">
        {user.links.map(link => <a href={link} className="text-blue-500 underline">{link}</a>)}
      </div>
    )}
  </div>
</ProfileHeader>

	•	Full-width, centered layout
	•	Shows:
	•	Profile picture
	•	Name, @username
	•	Bio (multi-line)
	•	External links

⸻

3. Filter Dropdown (Optional)

If a user has mixed content types:

<DropdownMenu
  items={["All", "Videos", "Shorts", "Photos", "Threads", "Audio"]}
  onSelect={setFilter}
/>

	•	Applies filter to user’s personal feed below

4. Reused Feed Card System

<PostFeed posts={filteredUserPosts}>
